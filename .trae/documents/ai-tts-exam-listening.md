# Plan: Browser speechSynthesis for HSK 4 Mock Exam Listening

## Summary

Use the **browser's built-in `window.speechSynthesis` API** (Web Speech API) for listening audio in the HSK 4 Mock Exam. This is what the current code already does, but I need to **improve it** so it works reliably and picks the best available Chinese voice.

User explicitly requested:
- Use browser speech (no Google TTS, no AWS Polly, no Azure, no third-party)
- The text-to-speak is already generated by the AI / by the word's `example_sentences` — we just need to **speak that text aloud using the device's built-in Chinese TTS voice**

The "AI generation" part of the pipeline is already done — every listening question in [src/services/exam.service.ts](file:///e:/PYTHON%20PROJECT%20UNI/MY%20HSK%204/hsk-vocab-app/src/services/exam.service.ts) already has an `audioText` field containing a real Chinese sentence. The only thing to improve is **voice selection and reliability** of the browser TTS.

## Current State Analysis

**ListeningPlayer.tsx** (current implementation) — [src/components/exam/ListeningPlayer.tsx](file:///e:/PYTHON%20PROJECT%20UNI/MY%20HSK%204/hsk-vocab-app/src/components/exam/ListeningPlayer.tsx):

```ts
const u = new SpeechSynthesisUtterance(text)
u.lang = 'zh-CN'
u.rate = speed
u.pitch = 1
u.onend = () => setPlaying(false)
u.onerror = () => setPlaying(false)
setPlaying(true)
window.speechSynthesis.speak(u)
```

**Problems with the current code:**

1. **No voice selection.** `u.lang = 'zh-CN'` only hints to the browser. On most desktops the browser falls back to a missing-voice, silent, or English voice. It should **explicitly pick a `SpeechSynthesisVoice` with `voice.lang.startsWith('zh')`**.
2. **Voices load asynchronously.** On Chrome/Edge, `window.speechSynthesis.getVoices()` returns `[]` until the `voiceschanged` event fires. If the user lands on a listening question before voices are loaded, the speak call will silently use a default/English voice.
3. **Speaking twice.** Because we don't await voices, the first `speak()` sometimes fires before the best voice is selected.
4. **No user feedback when no Chinese voice is available.** On iOS Safari, if the user has never enabled a Chinese voice in Settings → Accessibility → Spoken Content, the call silently fails.
5. **Long texts may be truncated.** `SpeechSynthesisUtterance` has a soft limit (~200 chars in Chrome). Long AI-generated listening passages need to be split on sentence boundaries.

## Proposed Changes

### 1. New file: `src/services/speech.service.ts`

A small, dedicated module that:

- Loads the best available Chinese voice asynchronously (waits for `voiceschanged`)
- Splits long Chinese text into sentence-sized chunks
- Exposes one function: `speakChinese(text, options?) => Promise<void>`
- Handles the "no Chinese voice available" case by throwing a clear error so the UI can show a helpful message

**Why:** isolates the Web Speech API quirks from React component logic, makes it testable, and means other future features (e.g. AI chat pronunciation, vocabulary word pronunciation) can reuse the same battle-tested speak() function.

**Implementation:**

```ts
// src/services/speech.service.ts
let cachedVoice: SpeechSynthesisVoice | null = null
let voicesReady: Promise<void> | null = null

/** Wait until the browser's voice list is populated. */
export function voicesLoaded(): Promise<void> {
  if (typeof window === 'undefined' || !('speechSynthesis' in window)) {
    return Promise.resolve()
  }
  if (cachedVoice) return Promise.resolve()
  if (voicesReady) return voicesReady
  voicesReady = new Promise<void>((resolve) => {
    const check = () => {
      const voices = window.speechSynthesis.getVoices()
      if (voices.length > 0) {
        cachedVoice = pickBestChineseVoice(voices)
        resolve()
      }
    }
    check()
    if (!cachedVoice) {
      window.speechSynthesis.addEventListener('voiceschanged', check, { once: true })
      // Some browsers never fire voiceschanged; fall back after 1s
      setTimeout(() => {
        if (!cachedVoice) {
          cachedVoice = pickBestChineseVoice(window.speechSynthesis.getVoices())
          resolve()
        }
      }, 1000)
    }
  })
  return voicesReady
}

function pickBestChineseVoice(voices: SpeechSynthesisVoice[]): SpeechSynthesisVoice | null {
  // Prefer high-quality local voices in this order:
  const preferred = [
    /Microsoft Xiaoxiao.*Chinese/i,  // Edge / Windows
    /Microsoft Yunyang.*Chinese/i,
    /Microsoft Yaoyao.*Chinese/i,
    /Tingting/i,                      // macOS
    /Sin-Ji/i,                        // macOS
    /Mei-Jia/i,                       // macOS
    /zh-CN/i,                         // Any zh-CN
    /zh_CN/i,
    /zh-Hans/i,
    /zh/i,                            // any Chinese voice
  ]
  for (const re of preferred) {
    const hit = voices.find((v) => re.test(v.name) || re.test(v.lang))
    if (hit) return hit
  }
  return null
}

/** Split long Chinese text into chunks ≤200 chars at sentence boundaries. */
function chunkText(text: string): string[] {
  if (text.length <= 200) return [text]
  const sentences = text.split(/(?<=[。！？!?\.])\s*/)
  const chunks: string[] = []
  let buf = ''
  for (const s of sentences) {
    if ((buf + s).length > 200 && buf) {
      chunks.push(buf.trim())
      buf = s
    } else {
      buf += s
    }
  }
  if (buf.trim()) chunks.push(buf.trim())
  return chunks
}

export interface SpeakOptions {
  rate?: number   // 0.1–10, default 1
  pitch?: number  // 0–2, default 1
}

export interface SpeakResult {
  ok: boolean
  hasChineseVoice: boolean
  error?: string
}

export async function speakChinese(text: string, options: SpeakOptions = {}): Promise<SpeakResult> {
  if (typeof window === 'undefined' || !('speechSynthesis' in window)) {
    return { ok: false, hasChineseVoice: false, error: 'speechSynthesis_not_supported' }
  }
  await voicesLoaded()
  window.speechSynthesis.cancel()
  const voice = cachedVoice
  if (!voice) {
    return { ok: false, hasChineseVoice: false, error: 'no_chinese_voice_installed' }
  }
  const chunks = chunkText(text)
  for (const chunk of chunks) {
    const u = new SpeechSynthesisUtterance(chunk)
    u.voice = voice
    u.lang = voice.lang || 'zh-CN'
    u.rate = options.rate ?? 1
    u.pitch = options.pitch ?? 1
    window.speechSynthesis.speak(u)
  }
  return { ok: true, hasChineseVoice: true }
}

export function stopSpeaking(): void {
  if (typeof window !== 'undefined' && 'speechSynthesis' in window) {
    window.speechSynthesis.cancel()
  }
}

export function hasChineseVoice(): boolean {
  return cachedVoice !== null
}
```

### 2. Refactor `src/components/exam/ListeningPlayer.tsx`

Replace the direct `speechSynthesis` call with `speakChinese` from the new service. Show a helpful message when no Chinese voice is installed (iOS / Linux users who haven't installed a voice).

**Changes:**

- Import `speakChinese, stopSpeaking` from `@/services/speech.service`
- Replace the inline `speak()` function with a call to `speakChinese(text, { rate: speed })`
- On mount, call `voicesLoaded()` early so voices are ready before the first auto-play
- Add a state `error: string | null` so we can show a one-line message: "Your device doesn't have a Chinese voice installed. Enable one in Settings → Accessibility → Spoken Content (iOS) or install Chinese language pack (Windows)."
- Keep the same `Preparing audio...` and `Replay audio` UI but add a small "fallback" message when no Chinese voice is available

**Refactored component (sketch):**

```tsx
import { useEffect, useRef, useState } from 'react'
import { Volume2, Loader2, AlertCircle } from 'lucide-react'
import { speakChinese, stopSpeaking, voicesLoaded } from '@/services/speech.service'

interface Props {
  text: string
  autoPlay?: boolean
  speed?: number
}

export default function ListeningPlayer({ text, autoPlay = true, speed = 1 }: Props) {
  const [playing, setPlaying] = useState(false)
  const [ready, setReady] = useState(false)
  const [error, setError] = useState<string | null>(null)
  const spokenRef = useRef(false)

  // Pre-warm the voice list so the first speak picks the best Chinese voice
  useEffect(() => {
    voicesLoaded().then(() => setReady(true))
  }, [])

  const handleSpeak = async () => {
    setError(null)
    const res = await speakChinese(text, { rate: speed })
    if (!res.ok) {
      setError(res.error === 'no_chinese_voice_installed'
        ? 'No Chinese voice on this device — install one in your OS settings.'
        : 'Browser does not support speech synthesis.')
      return
    }
    setPlaying(true)
    // onend handler in the service closes the playing flag via the
    // 'boundary'/'end' events on the last utterance; we approximate
    // by clearing after the chunk's expected duration.
    const wordsPerSec = 2.5 * speed
    const durationMs = Math.max(2000, (text.length / wordsPerSec) * 1000)
    setTimeout(() => setPlaying(false), durationMs)
  }

  // Auto-play once
  useEffect(() => {
    if (!autoPlay || spokenRef.current || !ready) return
    spokenRef.current = true
    const t = setTimeout(handleSpeak, 300)
    return () => clearTimeout(t)
    // eslint-disable-next-line react-hooks/exhaustive-deps
  }, [ready, autoPlay])

  // Cleanup on unmount
  useEffect(() => () => stopSpeaking(), [])

  if (!ready) {
    return (
      <div className="flex items-center gap-2 text-ink-500 dark:text-ink-400 text-sm">
        <Loader2 className="w-4 h-4 animate-spin" />
        Preparing audio...
      </div>
    )
  }

  if (error) {
    return (
      <div className="flex items-center gap-2 text-amber-600 dark:text-amber-400 text-xs">
        <AlertCircle className="w-4 h-4" />
        <span>{error}</span>
      </div>
    )
  }

  return (
    <button
      onClick={handleSpeak}
      disabled={playing}
      className="flex items-center gap-2 px-4 py-2.5 rounded-xl bg-teal-500 hover:bg-teal-600 disabled:opacity-70 text-white font-semibold text-sm transition-colors"
    >
      <Volume2 className="w-4 h-4" />
      {playing ? 'Playing...' : 'Replay audio'}
    </button>
  )
}
```

## Assumptions & Decisions

- **Best Chinese voice selection is heuristic-based.** We rank voices by quality (Microsoft Xiaoxiao / Tingting / Mei-Jia are higher quality than the generic `zh-CN` fallback). On most modern phones and desktops, this picks a natural-sounding voice automatically.
- **iOS limitation documented.** If the user hasn't enabled a Chinese voice in iOS settings, we show a one-line hint rather than failing silently. The user can then either install a voice (iOS) or install a Chinese language pack (Windows), and the app will start working.
- **Long passages are chunked** at sentence boundaries (。！？!?.) because Chrome's Web Speech API silently truncates utterances over ~200 chars.
- **No new dependency, no env vars, no vercel.json changes** — pure browser API.
- **The text-to-speak is unchanged.** The AI is still responsible for generating the script. The browser just renders it as audio.
